1) Прочитать про клиент-серверную архитектуру
Клиент-сервер - вычислительная или сетевая архитектура, в которой задания (запросы) распределяются между клиентами (заказчиками услуг) и серверами (поставщиками услуг). Фактически клиент и сервер - это программное обеспечение.
Примеры клиента: e-mail клиента, web browser, десктопное приложение и др.
Примеры сервера: любой компьютер или телефон, на котором запущена программа, которая будет принимать запросы и отправлять ответы.
2) Что такое HTTP и HTTPS
HTTP (HyperText Transport Protocol) - протокол прикладного уровня передачи данных. HTTPS (HyperText Transport Protocol) - это расширение протокола HTTP для поддержки шифрования в целях повышения безопасности.
HTTPS не является отдельным протоколом передачи данных, а представляет собой расширение протокола  HTTP с надстройкой шифрования. Данные, передаваемые по протоколу HTTP не защищены. HTTPS обеспечивает конфиденциальность информации путем ее шифрования. 
3) HTTP методы
Метод HTTP - последовательность символов, кроме управляющих и разделителей, указывающая на основную операцию над ресурсом. Обычно метод представляет собой короткое английское слово, написанное заглавными буквами. Название метода чувствительно к регистру.
Основные методы:
GET используется для получения или чтения данных. Запросы с использованием этого метода могут только получать данные.
HEAD запрашивает ресурс как и GET, но ответ приходит без тела 
PUT используется для обновления ресурса, путем замены данных запроса.
POST используется для создания нового ресурса или отправки сущности к определенному ресурсу.
DELETE удаляет данные.
OPTIONS используется для описания параметров соединения с ресурсом.
PATCH используется для частичного изменения ресурса.
4) HTTP статус коды сервера
Часть первой строки ответа сервера при запросах по протоколу HTTP. Первая цифра указывает класс состояния:
1хх: Informational
2xx: Success
3xx: Redirection
4xx: Client Error
5xx: Server Error

7) Что такое API
API - это набор готовых классов, процедур, функций, структур и констант, предоставляемых приложением (библиотекой, сервисом) для использования во внешних программных продуктах.
8) Что такое ендпоинты
Endpoint представляет собой некий шлюз, который соединяет серверные процессы приложения с внешним интерфейсом. Простыми словами это адрес, на который отправляются сообщения.
9) URL (URI, URL, URN)
URI - Uniform Resource Identifier (унифицированный идентификатор ресурса)
Обобщенное понятие идентификации ресурса. Можно считать что: URI = URL или URI = URN или URI = URL + URN
URL - Uniform Resource Locator (унифицированный определитель местонахождения ресурса)
Возник самый первый и закрепился как синоним веб-адреса. Определяет местонахождение ресурса в сети и способ его извлечения.
URN - Unifrorm Resource Name (унифицированное имя ресурса)
Неизменяемая последовательность символов, определяющая только имя некоторого ресурса.
10) Идемпотентные HTTP методы
Метод HTTP является идемпотентным, если повторный идентичный запрос, сделанный один или несколько раз, имеет один и тот же эффект, не изменяющий состояние сервера. Данный метод не должен иметь никаких побочных эффектов, кроме сбора информации. Методы GET, HEAD, PUT и DELETE являются идемпонентными. Все безопасные методы являются идемпотентными.
11) Безопасные HTTP методы
Метод НТТР является безопасным, если он не меняет состояние сервера. Методы GET, HEAD или OPTIONS являются безопасными.
12) Идентификация, Аутентификация, Авторизация
Идентификация - это процедура распознания субъекта по его идентификатору.
Аутентификация - это процедура проверки подлинности.
Авторизация - предоставление доступа к какому-либо ресурсу.
13) Что такое IP
Internet Protocol - маршрутизируемый сетевой протокол. IP-адрес - уникальный числовой идентификатор устройства в компьютерной сети, работающей по протоколу IP.
14) Что такое октеты в DNS
Строки октетов используются при адресации по IP, в которой четыре байта адреса IPv4 состоят из четырех октетов. Веб-клиенты используют потоки октетов при работе с несколькими типами двоичных файлов и когда они не могут распознать тип по имени файла или принять какой-либо один конкретный формат. Браузеры часто предлагают пользователю определить тип файла потока октетов, сохранив файл с определенным расширением имени файла.
15) Что такое порт, сколько портов у Linux сервера
Порт сервера - это цифровой идентификатор (набор цифр до 16 Бит) программы, которая используется для создания определенного интернет соединения привязанного к IP-адресу. HTTP использует порт 80, а HTTPS - порт 443. Linux-сервер може поддерживать 65536 портов.
16) Уровни OSI
Open Systems Interconnection model - сетевая модель, используемая для того, чтобы разбить всех участников сети на некоторые категории (уровни) взаимодействия с другими протоколами и со средой передачи данных.
Состоит из 7 уровней:
1 Phisical
2 Data Link
3 Network
4 Transport
5 Session
6 Presentation
7 Application
17) Хедеры http запросов
Заголовки HTTP позволяют клиенту и серверу отправлять дополнительную информацию с HTTP запросом или ответом. В HTTP-заголовке содержится не чувствительное к регистру название, а затем после (:) непосредственно значение.
